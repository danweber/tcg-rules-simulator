Rule Simulator v0.7.12

Don't pass this around yet. I want to keep the audience small because it's not ready for a lot of people. This is a technology demonstration.

If you try to break the software by messing with the inputs you'll succeed and it won't be interesting. If you breaking the game logic by playing a weird set of cards, that is interesting.

This is only ST15 versus ST16 (with two added cards, see below).

I hate the UI but it gets the job done for now. I'd love to pass UI development off to someone else. Someone with more experience would have something better in less than a day.

    Quick start
  • You might have to press "Send" or "Tick" to first populate the game after loading the page.
  • For now all user-input happens in the UI box at the top.
  • If you need to choose something (which effect, which target, to block) it'll be in that same list.
  • After some moves (anything that involves effects, or attacking, the game will pause on purpose. The point of the rule simulator is to be able to stop at various points, rewind, and see what's happening, so this was built in from the start. For now, just press "tick" to move the game forward.
  • Basically, if "TICK" is blue, press it.
  • The egg zone will have a pixel-art background, if it exists. The trash will have a black background, if it exists.
  • Because ST15 versus ST16 has no interruptive effects, and those were very important to test, I made up Armormon (Agumon with Armor Purge), and then gave Matt Piercemon (Garurumon that pierces) for balance. OC DO NOT STEAL!!1one

Got that? Great, get started!

If you want to try, open this in one tab and this in another. There's no mulligan right now.

Bugs

    Known bugs
  • Decking out doesn't end the game.
  • Armor-purge versus armor-purge only lets one player save mon
  • User isn't prompted on all optional effects
  • Lament of Friendship can target cards in hand.
  • On-Deletion effects will still activate if the card is no longer in the trash
  • "Must attack" effect can only attack opponent's digimon, not the player, and the digimon will still suspend even if it can't attack
  • MetalGarurumon can discard more than 3 cards for his when-digivolve effect
    Bugs I think are fixed but might still be around
  • Sometimes things in the egg zone or the trash get activated or targetted.
  • Once-per-turn may or may not be working
    Bugs you should report
  • Cards in ST15 versus ST16 not doing the right thing
  • Accidental UI lockups.
    What not to report
  • How a cheating player can cheat.
  • Situations that you cannot get into in an ST15-versus-ST16 match.
    Reproducing issues
  • Each gid uses the same random seed at same start for debugging purposes. If you want it to be really random, start the gid with "rand". If you want both players to have the same seed (meaning they'll get a similar mix of levels, tamers, and options) prefix the seed with "same".
  • You can reset the game by putting "reset" in the instruction box in the top left and hitting Send.

Other stuff

    Design decisions you may or may not notice
  • You are prompted to activate an event where the cost can't be paid. For example, suspending Matt to gain a memory, even if Matt is already suspended. This should be changed to not being able to choose at all, although right now it's helping to test the tried-to-pay-the-cost-but-failed logic.
  • You can also choose effects that have used up their once-per-turn, but the effect immediately fails.
  • The game gives away if your opponent could choose to blast digivolve. It's easy to make it just always pause. Maybe I'll do that.
  • Right now color is ignored except in two cases: first, making sure you have a color for playing an option. The second is for Solarmon's searcher effect, because I wanted it to be able to fail during testing.
  • If an effect has 0 or 1 targets, the game will autochoose and continue. This was a design choice and it might be wrong if people want to notice what's going on.
  • Your option card is in the trash immediately after activation. It should actually hang around null space until it's done.
  • Retaliation might activate on mutual battle death. It won't do anything, you won't notice unless youu watch closely. But it shouldn't activate at all.
    More details:
  • For now the game somewhat depends on you knowing what's going on. If Agumon's effects activate at start of main I hope you know what that means. This can certainly be made better but other tasks were more important.
  • I think I've gotten rid of the UI bug where the wrong player can control the game during questions, leading to weird situations, but it might still be around.
  • If you're on phone the logs are probably not visible because they're on the right. Landscape will show them but you'll end up scrolling a lot.
  • The UI could be a separate project. The API is pretty easy and the server does all the work, even telling you what cards move where.
